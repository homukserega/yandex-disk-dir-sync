# Синхронизация локальной папки с Яндекс.Диском

Проект представляет собой Python-приложение, которое автоматически синхронизирует указанную локальную директорию с выбранной папкой на Яндекс.Диске.
Синхронизация выполняется в режиме реального времени (с периодической проверкой каждые 10 секунд) – отслеживаются:

    новые файлы (загружаются на Диск);

    удалённые файлы (удаляются на Диске);

    изменённые файлы (перезаписываются на Диске на основе времени модификации).

📋 Требования

    Python 3.8+

    Учётная запись Яндекс.Диска

    OAuth-токен Яндекс.Диска с правами на запись/чтение

    Docker (опционально, для контейнеризации)

⚙️ Установка и настройка
1. Клонирование репозитория
bash

git clone <url-репозитория>
cd <папка-проекта>

2. Переменные окружения

Создайте файл .env на основе шаблона env-template:
bash

cp env-template .env

Отредактируйте .env, указав свои значения:

Переменная	Описание

YANDEX_TOKEN	OAuth-токен Яндекс.Диска (получить в полигоне API)

YANDEX_DISK_PATH	Путь к папке на Яндекс.Диске, с которой выполняется синхронизация (например, backup или sync_folder)

HOST_PATH	Путь к локальной папке для синхронизации на вашем ПК. Например: /home/user/Downloads

LOG_PATH	Путь к папке для логов (по умолчанию ./logs) – создаётся автоматически

    Важно: YANDEX_DISK_PATH – это имя папки на Диске, не полный путь. Если папка не существует, приложение завершится с ошибкой.

🐳 Запуск с Docker (рекомендуемый способ)

Проект поставляется с Dockerfile и docker-compose.yaml для простого развёртывания.
Сборка и запуск
bash

> docker-compose up

Или в фоновом режиме

> docker compose up -d

Контейнер будет автоматически перезапускаться при сбоях (политика restart: unless-stopped).
Логи пишутся в папку ./logs на хосте (монтируется в контейнер).
Остановка
bash

> docker-compose down
>> в случае ошибки ввода пользовательских данных(см. логи) - исправьте .env - перезапустите программу

Просмотр смонтированный локально логов
bash

> docker volume inspect yandex-disk-dir-sync_logs_volume

После чего будет выведен путь в виде значения, указанного в <b>Mountpoint</b>

Название файла: <b>ya-disk-sync.log</b>

🔁 Принцип работы

    При старте программа получает список файлов из удалённой папки Яндекс.Диска (с их временем модификации).

    Сканирует локальную папку HOST_PATH и получает список файлов с их mtime.

    Сравнивает два списка:

        Файлы, которых нет локально → удаляются на Яндекс.Диске.

        Файлы, которых нет на Диске → загружаются.

        Файлы с разным временем модификации → перезаписываются на Диске (загружаются заново).

    После каждого цикла обновляется удалённый список, и всё повторяется каждые 10 секунд.

При загрузке файлов на Диск сохраняется их исходное mtime в custom_properties, чтобы при следующей синхронизации корректно определять изменения.
📝 Логирование

    Логи выводятся в консоль с цветным форматированием (уровень INFO).

    Также логи сохраняются в файл контейнера logs/ya-disk-sync.log с ротацией (максимальный размер 100 МБ, хранение 30 дней).

Пример записи в логе:
text

ya-disk-sync 2026-07-19 10:15:32,123 INFO Файл 'document.txt' успешно загружен!

⚠️ Обработка ошибок

    Неверный OAuth-токен – программа завершается с сообщением об ошибке авторизации.

    Несуществующая папка на Яндекс.Диске – программа завершается с ошибкой.

    Сетевые сбои – логируются, но цикл синхронизации продолжается (повтор через 10 секунд).

    Ошибки доступа к файлам (нет прав, файл занят) – логируются, но работа программы не останавливается.

Все ошибки обрабатываются с использованием кастомных исключений (FileSyncError, UnauthorizedError), а удобочитаемые описания формируются через функцию map_error_type.
